# JavaScript — `JSON.stringify` with `undefined` & `null`

## Core Concept: JSON Has Its Own Data Types

In JavaScript, we use `undefined` and `null` to represent "no value" — but they behave **very differently** when passed through `JSON.stringify()`. Understanding _why_ requires knowing what data types JSON actually supports.

---

## 1. The Basics — What is `undefined`?

`undefined` means a variable has been **declared but never assigned a value**. JavaScript creates this automatically.

```javascript
let x
console.log(x) // undefined

const obj = { a: undefined }
console.log(obj.a) // undefined
```

> ⚠️ **Critical Point**: `undefined` is a **JavaScript-only** concept. It does **not exist** as a data type in the JSON specification.

---

## 2. The Basics — What is `null`?

`null` is a value you **explicitly assign** to signal "intentionally empty". Unlike `undefined`, it is something you choose to set.

```javascript
let y = null
console.log(y) // null
```

> ✅ **Critical Point**: `null` **does exist** as a valid JSON data type. JSON was designed to handle `null`.

---

## 3. What is JSON?

**JSON (JavaScript Object Notation)** is a text-based data format used for storing and sending data. It has its **own strict set of allowed types** — not everything in JavaScript is valid in JSON.

### Valid JSON Data Types

| Type    | Example              |
| ------- | -------------------- |
| String  | `"hello"`            |
| Number  | `42`, `3.14`         |
| Boolean | `true`, `false`      |
| null    | `null`               |
| Object  | `{ "key": "value" }` |
| Array   | `[1, 2, 3]`          |

```
JSON Supported Types:
  ✅ string
  ✅ number
  ✅ boolean
  ✅ null
  ✅ object
  ✅ array
  ❌ undefined   ← Does NOT exist in JSON
  ❌ function    ← Does NOT exist in JSON
  ❌ Symbol      ← Does NOT exist in JSON
```

---

## 4. The Problem: `undefined` in Objects

### What Happens?

```javascript
JSON.stringify({ a: undefined, b: 'Binary dev' })

// Output: '{"b":"Binary dev"}'
// ⚠️ The key "a" is COMPLETELY GONE from the result
```

### Chain of Events

```
Step 1: JSON.stringify encounters key "a" with value undefined
Step 2: It checks — is undefined a valid JSON type?
Step 3: Answer is NO → JSON has no way to represent this
Step 4: The key "a" and its value are silently DROPPED
Step 5: Only "b" survives in the output
```

### Visual Representation

```
Input Object:
┌──────────────────────────────┐
│  a  →  undefined             │  ← ❌ Not valid in JSON → KEY DELETED
│  b  →  "Binary dev"          │  ← ✅ Valid string → KEPT
└──────────────────────────────┘
              ↓
        JSON.stringify
              ↓
Output JSON String:
┌──────────────────────────────┐
│  b  →  "Binary dev"          │
└──────────────────────────────┘
Result: '{"b":"Binary dev"}'
```

### Compare with `null`

```javascript
JSON.stringify({ a: null, b: 'Binary dev' })

// Output: '{"a":null,"b":"Binary dev"}'
// ✅ Key "a" is preserved with value null
```

`null` is kept because JSON natively understands it.

---

## 5. The Problem: `undefined` in Arrays

### What Happens?

```javascript
JSON.stringify([undefined, 'Binary dev'])

// Output: '[null,"Binary dev"]'
// 🔄 undefined becomes null — it is NOT deleted
```

### Why Arrays Behave Differently from Objects

Arrays are **index-based**. Every element has a positional address in memory. Deleting an element would **shift all subsequent indexes**, corrupting your data.

```
Array:  [ undefined, "Binary dev" ]
Index:      [0]           [1]

If we DELETE index [0]:
  → "Binary dev" shifts from [1] to [0]
  → Any code reading index [1] now gets undefined
  → DATA IS CORRUPTED ❌

So instead, JSON substitutes null as a placeholder:
  → [ null, "Binary dev" ]
  → [0] is null, [1] is still "Binary dev"
  → Array length and positions are preserved ✅
```

### Visual Representation

```
Input Array:
┌───────┬──────────────────┐
│ [0]   │  undefined       │  ← ❌ Not valid in JSON, but index must exist
│ [1]   │  "Binary dev"    │  ← ✅ Valid string
└───────┴──────────────────┘
              ↓
        JSON.stringify
              ↓
Output Array:
┌───────┬──────────────────┐
│ [0]   │  null            │  ← 🔄 Replaced with null (placeholder)
│ [1]   │  "Binary dev"    │  ← ✅ Unchanged, index preserved
└───────┴──────────────────┘
Result: '[null,"Binary dev"]'
```

## 6. Side-by-Side Comparison

| Input                       | Output                | What Happened                              |
| --------------------------- | --------------------- | ------------------------------------------ |
| `{ a: undefined, b: "hi" }` | `{"b":"hi"}`          | Key `a` silently dropped                   |
| `{ a: null, b: "hi" }`      | `{"a":null,"b":"hi"}` | Key `a` preserved — `null` is valid JSON   |
| `[undefined, "hi"]`         | `[null,"hi"]`         | `undefined` swapped for `null` placeholder |
| `[null, "hi"]`              | `[null,"hi"]`         | No change — `null` is valid JSON           |

## 7. The Fix — `undefinedToNull()` Helper

### The Problem

When sending data to an API, you may need **all keys preserved** even if their values are `undefined`. A dropped key can break things on the receiving end.

### The Solution

```javascript
function undefinedToNull(data) {
  return JSON.parse(
    JSON.stringify(data, (key, value) => {
      if (value === undefined) {
        return null
      }
      return value
    }),
  )
}
```

### Step-by-Step Code Breakdown

#### Step 1 — The Replacer Function

`JSON.stringify` accepts an optional **replacer** as its second argument. This function runs for **every key-value pair**, letting you control what gets serialised.

```javascript
JSON.stringify(data, (key, value) => {
  //                   ↑     ↑
  //               key name  current value being processed
})
```

#### Step 2 — Intercept `undefined` Before JSON Sees It

```javascript
;(key, value) => {
  if (value === undefined) {
    return null // ← Return null INSTEAD of undefined
  }
  return value // ← Everything else passes through unchanged
}
```

By returning `null`, we hand JSON a value it **can understand**, preventing the key from being dropped.

#### Step 3 — `JSON.parse` to Get Back an Object

`JSON.stringify` produces a string. `JSON.parse` converts it back to a usable JavaScript object.

```javascript
return JSON.parse( JSON.stringify(...) );
//       ↑
//  Converts the JSON string back into a proper JS object
```

### Execution Trace

```javascript
undefinedToNull({ a: undefined, b: "Binary dev" })

Replacer runs on key "a":
  value === undefined → return null
  JSON now sees: { a: null, b: "Binary dev" }

JSON.stringify produces:
  '{"a":null,"b":"Binary dev"}'

JSON.parse produces:
  { a: null, b: "Binary dev" }  ✅ Key "a" is preserved!
```

### Usage Example

```javascript
const user = { name: 'Alice', age: undefined, city: 'Bangalore' }

// Without fix:
console.log(JSON.stringify(user))
// '{"name":"Alice","city":"Bangalore"}'  ← age is LOST

// With fix:
console.log(undefinedToNull(user))
// { name: "Alice", age: null, city: "Bangalore" }  ← age is PRESERVED ✅
```

## 8. Key Takeaways

| Concept                 | Summary                                                                              |
| ----------------------- | ------------------------------------------------------------------------------------ |
| **`undefined` in JSON** | Does not exist — gets dropped from objects, replaced with `null` in arrays           |
| **`null` in JSON**      | Fully valid — always preserved exactly as-is                                         |
| **Object behaviour**    | `undefined` values cause the entire key to be silently removed                       |
| **Array behaviour**     | `undefined` becomes `null` to preserve index positions                               |
| **Replacer function**   | Second argument to `JSON.stringify` — intercepts values before serialisation         |
| **`undefinedToNull()`** | Custom helper to safely convert all `undefined` values to `null` before stringifying |

---

## Real-World Analogy

Think of JSON like an **international postal form**. It has a fixed set of fields it understands. If you try to write something in a language it doesn't support (like `undefined`), it ignores that field and moves on. But `null` is a recognised word that means "intentionally blank" — so it gets written down just fine.

---

## Summary

`JSON.stringify()` behaves differently for `undefined` depending on context:

- **In objects** → the key is completely **deleted** (JSON can't represent `undefined`)
- **In arrays** → `undefined` is **replaced with `null`** (to preserve index positions)

Use a **replacer function** to intercept and convert `undefined → null` when you need all keys preserved.

**Remember**: JSON has its own type system — when in doubt, check whether your value type is on JSON's supported list!
