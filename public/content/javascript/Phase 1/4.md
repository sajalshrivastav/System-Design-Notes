# JavaScript — Function Composition, `pipe()` & `compose()`

## Core Concept: Functions as Building Blocks

In JavaScript, functions are **first-class citizens** — they can be passed around, stored, and combined like any other value. **Function Composition** is a technique that leverages this to build powerful logic by chaining small, focused functions together.

---

## 1. What is Function Composition?

**Function Composition** is the process of combining two or more functions to produce a **new function**, where the output of one function becomes the input of the next.

```javascript
// Without composition — nested, hard to read:
const result = square(addOne(double(3)))

// With composition — clean, readable pipeline:
const transform = compose(square, addOne, double)
const result = transform(3)
```

### Simple Example

```javascript
const double = (x) => x * 2
const addOne = (x) => x + 1
const square = (x) => x * x

// Manual nesting:
const result = square(addOne(double(3)))
// double(3) = 6 → addOne(6) = 7 → square(7) = 49

console.log(result) // 49
```

### Visualising the Flow

```
Input: 3
  │
  ▼
[double]  →  3 * 2 = 6
  │
  ▼
[addOne]  →  6 + 1 = 7
  │
  ▼
[square]  →  7 * 7 = 49
  │
  ▼
Output: 49
```

Think of it like an **assembly line** — raw material enters one end and passes through each station until you get a finished product.

```
Raw Input → [Function A] → [Function B] → [Function C] → Final Output
```

---

## 2. Why Use Function Composition?

### The Problem Without It

```javascript
// Imagine doing all this inline, every time you need it:
const username = addPrefix(removeSpaces(toLowerCase(trim('  Binary Dev  '))))

// What if the logic changes? You have to hunt down every place you used this.
// What if you need it in 10 different files?
```

### The Solution

Break logic into **small, reusable, single-purpose functions** and compose them together.

```javascript
const createUsername = pipe(trim, toLowerCase, removeSpaces, addPrefix)

// Now you have a named, reusable function.
// Change the pipeline in one place, it updates everywhere.
createUsername('  Binary Dev  ') // "user-binary-dev"
```

### Benefits Table

| Benefit                | Explanation                                                             |
| ---------------------- | ----------------------------------------------------------------------- |
| ✅ **Readability**     | Describes the data flow step by step, like reading a recipe             |
| ✅ **Reusability**     | Small functions can be mixed, matched, and reused across your codebase  |
| ✅ **Testability**     | Each small function is easy to test in isolation                        |
| ✅ **Maintainability** | Add or remove a step by just adding/removing a function in the pipeline |
| ✅ **No Side Effects** | Each function takes input and returns output — pure, predictable logic  |

---

## 3. Types of Function Composition

There are two common implementations of function composition in JavaScript:

| Type        | Order of Execution | Reads Like                      |
| ----------- | ------------------ | ------------------------------- |
| `pipe()`    | Left → Right       | Natural reading order           |
| `compose()` | Right → Left       | Mathematical notation `f(g(x))` |

Both achieve the **same result** — the only difference is the **order you list the functions**.

---

## 4. `pipe()` — Left to Right

### What It Does

`pipe()` takes a series of functions and returns a new function. When called, it passes the input through each function **from left to right** — the first function listed runs first.

```javascript
pipe(double, addOne, square)(3)
// → double runs first, then addOne, then square
// → 3 → 6 → 7 → 49
```

### Implementation

```javascript
const pipe =
  (...fns) =>
  (x) =>
    fns.reduce((acc, fn) => fn(acc), x)
```

### Line-by-Line Breakdown

```javascript
const pipe =
  (...fns) =>
  //            ↑
  //  (...fns) is a rest parameter.
  //  It collects ALL the functions you pass into an array.
  //  pipe(double, addOne, square)
  //    → fns = [double, addOne, square]

  (x) =>
    //  ↑
    //  pipe RETURNS A NEW FUNCTION that accepts the starting value.
    //  pipe(...)(3) → x = 3

    fns.reduce((acc, fn) => fn(acc), x)
//      ↑
//  .reduce() iterates over the fns array from LEFT TO RIGHT.
//  - acc = accumulated result (starts as x)
//  - fn  = current function being applied
//  - fn(acc) = call current function with the result so far
//  Starting value of acc = x (the initial input)
```

### How `reduce` Executes Step by Step

```
fns = [double, addOne, square]
x = 3

Pass 1:  acc = 3,  fn = double  →  double(3)  = 6
Pass 2:  acc = 6,  fn = addOne  →  addOne(6)  = 7
Pass 3:  acc = 7,  fn = square  →  square(7)  = 49

Final result: 49 ✅
```

### Usage

```javascript
const double = (x) => x * 2
const addOne = (x) => x + 1
const square = (x) => x * x

const transform = pipe(double, addOne, square)

console.log(transform(3)) // 49
console.log(transform(5)) // 5→10→11→121
```

## 5. `compose()` — Right to Left

### What It Does

`compose()` is identical to `pipe()` except it executes functions **from right to left** — the last function listed runs first. This mirrors how function composition is written in mathematics: `f(g(h(x)))`.

```javascript
compose(square, addOne, double)(3)
// → double runs first (rightmost), then addOne, then square
// → 3 → 6 → 7 → 49
```

### Implementation

```javascript
const compose =
  (...fns) =>
  (x) =>
    fns.reduceRight((acc, fn) => fn(acc), x)
```

### Line-by-Line Breakdown

```javascript
const compose =
  (...fns) =>
  //               ↑
  //  Same as pipe — collects all functions into an array.

  (x) =>
    //  Returns a new function accepting the starting value.

    fns.reduceRight((acc, fn) => fn(acc), x)
//       ↑
//  .reduceRight() is identical to .reduce() EXCEPT it
//  iterates from RIGHT TO LEFT (end of array to start).
//  This means the LAST function you listed runs FIRST.
```

### How `reduceRight` Executes Step by Step

```
fns = [square, addOne, double]
x = 3

(reduceRight starts from the END of the array)

Pass 1:  acc = 3,  fn = double  →  double(3)  = 6
Pass 2:  acc = 6,  fn = addOne  →  addOne(6)  = 7
Pass 3:  acc = 7,  fn = square  →  square(7)  = 49

Final result: 49 ✅
```

### Usage

```javascript
const double = (x) => x * 2
const addOne = (x) => x + 1
const square = (x) => x * x

const transform = compose(square, addOne, double)

console.log(transform(3)) // 49
console.log(transform(5)) // 5→10→11→121
```

## 6. `pipe()` vs `compose()` — Head to Head

Both of these produce **identical results** for the same logical sequence:

```javascript
pipe(double, addOne, square)(3) // 49
compose(square, addOne, double)(3) // 49
```

The difference is **only in how you write the function list**:

```
pipe:     pipe(FIRST_TO_RUN, ..., LAST_TO_RUN)
compose:  compose(LAST_TO_RUN, ..., FIRST_TO_RUN)
```

### Visual Comparison

```
pipe(double, addOne, square)(3)
─────────────────────────────────
  Read/execute order: LEFT → RIGHT

  3  →  [double]  →  [addOne]  →  [square]  →  49
         (runs 1st)   (runs 2nd)   (runs 3rd)


compose(square, addOne, double)(3)
─────────────────────────────────
  Read order: LEFT → RIGHT
  Execute order: RIGHT → LEFT

  3  →  [double]  →  [addOne]  →  [square]  →  49
         (runs 1st)   (runs 2nd)   (runs 3rd)
```

### Key Differences Table

| Feature         | `pipe()`                | `compose()`                   |
| --------------- | ----------------------- | ----------------------------- |
| Execution order | Left → Right            | Right → Left                  |
| First to run    | Leftmost function       | Rightmost function            |
| Reads like      | A to-do list (top-down) | Math notation `f(g(h(x)))`    |
| Uses            | `.reduce()`             | `.reduceRight()`              |
| When to prefer  | Most everyday use cases | When working in math/FP style |

## 7. Real-World Example — Text Processing Pipeline

```javascript
const pipe =
  (...fns) =>
  (x) =>
    fns.reduce((acc, fn) => fn(acc), x)

// Each function does ONE thing:
const trim = (str) => str.trim()
const toLowerCase = (str) => str.toLowerCase()
const removeSpaces = (str) => str.replace(/\s+/g, '-')
const addPrefix = (str) => `user-${str}`

// Compose them into a named, reusable pipeline:
const createUsername = pipe(trim, toLowerCase, removeSpaces, addPrefix)

console.log(createUsername('  Binary Dev  '))
// trim          → "Binary Dev"
// toLowerCase   → "binary dev"
// removeSpaces  → "binary-dev"
// addPrefix     → "user-binary-dev"  ✅
```

### Execution Trace

```
Input: "  Binary Dev  "

Step 1 — trim:
  "  Binary Dev  "  →  "Binary Dev"

Step 2 — toLowerCase:
  "Binary Dev"  →  "binary dev"

Step 3 — removeSpaces:
  "binary dev"  →  "binary-dev"

Step 4 — addPrefix:
  "binary-dev"  →  "user-binary-dev"

Output: "user-binary-dev" ✅
```

## 8. How They Differ from Normal Function Calls

```javascript
// Normal — executes immediately, result is a value:
const result = square(addOne(double(3)))
console.log(result) // 49

// Composed — creates a REUSABLE function first:
const transform = pipe(double, addOne, square)
// transform is a function, NOT a value yet

console.log(transform(3)) // 49
console.log(transform(10)) // 10→20→21→441
// The same pipeline, called multiple times ✅
```

> **Key Insight**: `pipe()` and `compose()` are **function factories**. They take functions as input and return a new function. The new function doesn't run until you call it with a value.

---

## 9. Key Takeaways

| Concept                  | Summary                                                                                   |
| ------------------------ | ----------------------------------------------------------------------------------------- |
| **Function Composition** | Combining functions so the output of one becomes the input of the next                    |
| **`pipe()`**             | Executes functions left → right; list them in the order they should run                   |
| **`compose()`**          | Executes functions right → left; list them in reverse order of execution                  |
| **`reduce()`**           | The engine behind `pipe()` — iterates left to right over the function array               |
| **`reduceRight()`**      | The engine behind `compose()` — iterates right to left over the function array            |
| **Rest params `...fns`** | Collects any number of functions into an array, making both utilities infinitely flexible |

---

## Real-World Analogy

Think of `pipe()` like a **water treatment plant**:

- Water enters at one end (your input)
- It passes through **Filter A → Filter B → Filter C** in order
- Clean water exits the other end (your output)

Each filter (function) does one job. You can add, remove, or reorder filters without rebuilding the whole plant.

---

## Quick Reference

```javascript
// pipe — left to right
const pipe =
  (...fns) =>
  (x) =>
    fns.reduce((acc, fn) => fn(acc), x)

// compose — right to left
const compose =
  (...fns) =>
  (x) =>
    fns.reduceRight((acc, fn) => fn(acc), x)

// Usage:
pipe(f, g, h)(x) === h(g(f(x))) // f runs first
compose(h, g, f)(x) === h(g(f(x))) // f still runs first
```

---

## Summary

Function Composition lets you build **complex logic from simple building blocks**. Instead of writing deeply nested function calls or long chains of intermediate variables, you describe a clear, readable pipeline.

- Use **`pipe()`** when you want to list functions in the order they run (most intuitive for everyday code)
- Use **`compose()`** when you prefer mathematical notation or are working in a functional programming style

**Remember**: Both `pipe()` and `compose()` return a **new function** — they don't execute immediately. Call the result with a value to run the pipeline!
